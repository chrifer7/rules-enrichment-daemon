# OpenShift Manifests - rules-enrichment-daemon

Estructura:
- `dev/` manifiestos para development
- `test/` manifiestos para testing

Nomenclatura aplicada:
- `is` -> ImageStream
- `bc` -> BuildConfig
- `d` -> Deployment
- sufijo de entorno: `dev` o `test`

Ejemplos:
- `rules-enrichment-daemon-dev-is`
- `rules-enrichment-daemon-dev-bc`
- `rules-enrichment-daemon-dev-d`
